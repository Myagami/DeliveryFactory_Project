□宅配便産業セット□

○概要
当アドオンはTwitter上でイカッピーさんが発言していた「通販を産業化出来ないか?」という発言を受けて通販産業という物を作っていたのですが
いつの間にか企画自体が自然消滅しており発想自体はかなり面白かったので当アドオンで配送部分を実装した上で
通販産業の部分は別途自分もしくは誰かに作ってもらおうと考えています。

当アドオン自体は単独でも使えるようになっておりますので導入していただいた上で宅配便をマップ内に走らせてください。
産業としての建物自体はただの箱になっております、箱積駅舎を組み合わせた上で使うとそれっぽく建物が作れるはずです。

○同梱物一覧と解説
Delivery_Office(Out) / 営業所(発送)
小口荷物をセンターに向けて発送(生産)します
Delivery_Office(In) / 営業所(配送)
センターからやってきた小口荷物を配送(消費)します

Delivery_DistrictBase(Out) / 地区物流センター(発送)
配下の営業所から送られてきた小口荷物を集め(消費)、別の物流センターもしくは幹線センターへ地区カーゴへ積み替えます(生産)
Delivery_DistrictBase(In) / 地区物流センター(配送)
やってきた地区カーゴを受け取り(消費)、配下にある営業所宛の小口荷物に振り分けます(生産)

Delivery_TrunkLine_Base(Out) /  主要物流センター(発送)
地区物流センターからやってきた地区カーゴを受け取り(消費)、他地区にある主要物流センター行く幹線カーゴへ積み替えます(生産)
Delivery_TrunkLine_Base(In) / 主要物流センター(配送)
他の地区物流センターからやってきた幹線カーゴを受け取り(消費)、配下にある地区物流センターへ行く地区カーゴへ積み替えます(生産)

---貨物一覧---
Delivery_Box(Small) / 小口荷物
営業所と地区物流センター間でやり取りされる貨物です
Delivery_Box(District) / 地区カーゴ
地区物流センター間もしくは地区物流センターと主要物流センター間でやり取りされる貨物です
Delivery_Box(Trunk) / 幹線カーゴ
主要物流センター間でやり取りされる貨物です

全て混載貨物扱いですので飛行機を使って輸送することも可能です、ぜひご利用ください。


○翻訳ファイルについて
同梱しておりますja.DeliveryFactory.tabを導入すると日本語訳化して表示可能となっております､ぜひご利用くださいませ

○基本的な輸送フロー
営業所(発送 / Out) → 地区センター(配送 / In) / この間を駅等で連絡(直結)できるようにする / 地区センター(発送 / Out) → 営業所(配送 / In) / 近距離地区内完結パターン
営業所(発送 / Out) → 地区センター(配送 / In) → 主要物流センター(発送 / Out)

↓ この間貨物列車等で継走

主要物流センター(配送 / In) → 地区センター(発送 / Out) → 営業所(配送 / In) / 遠距離輸送パターン

基本的な産業の接続はOutからInに繋ぐ事と営業所以外は地区センター同士や物流センターで繋げられるになります

○アドオン製作者向け
カーゴや小口荷物を生産もしくは消費する産業を制作すれば仕入れや出荷が擬似的に再現出来ます
既存の産業を当アドオンのフローに組み込む時は変換アダプタ(実装例として倉庫等)として作用する産業が必要です
